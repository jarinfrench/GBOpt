# Grain Boundary Optimization Example

This example demonstrates the use of GBOpt to optimize grain boundary (GB)
energy across six boundary types and three materials (Fe, Ni, Si) using both
genetic algorithm (GA) and Monte Carlo (MC) methods. These are the results
referenced in the accompanying manuscript.

## Directory structure

```
gb_optimization/
├── config/
│   ├── boundaries.py          # misorientation arrays for each boundary
│   ├── ga.toml                # GA hyperparameters
│   ├── mc.toml                # MC hyperparameters
│   └── literature_gbe.toml   # reference GBE values from literature
├── materials/
│   ├── Fe.toml                # lattice parameter, structure, potential info
│   ├── Ni.toml
│   └── Si.toml
├── lammps/
│   ├── lmp.in                 # LAMMPS input script
│   ├── min_strat.commands     # minimization strategy commands
│   ├── Fe/potential.setup
│   ├── Ni/potential.setup
│   └── Si/potential.setup
├── scripts/
│   ├── make_initial.py        # generate initial LAMMPS structures
│   ├── run_ga.py              # run GA optimization
│   ├── run_mc.py              # run MC optimization
│   ├── submit.py              # SLURM job submission helper
│   ├── find_min_gbe.py        # find minimum GBE for a single run
│   ├── extract_ga_data.py     # extract ga_data.json from pkl files
│   ├── plot_ga_comparison.py  # per-material GA panel plots
│   └── plot_mc_comparison.py  # per-material MC band plots
├── Fe/
│   └── {boundary}/
│       ├── best_structure.txt
│       ├── high_e_initial.txt   # high-energy restart structure (sigma5_310_STGB only)
│       ├── GA/
│       │   ├── standard/run*/ga_data.json
│       │   └── high_energy/run*/ga_data.json
│       └── MC/
│           ├── standard/run*/gbe_vals.txt
│           └── high_energy/run*/gbe_vals.txt
├── Ni/   (same structure as Fe)
└── Si/   (same structure as Fe)
```

> **Note:** `best_structure.txt` is not written automatically by any script.
> `find_min_gbe.py` *prints* the path to the
> minimum-GBE dump file found within the run directory; it does not copy it.
> To create `best_structure.txt`, identify the best structure with that
> script and copy the file manually.

## Track 1 — Reproduce manuscript figures

All optimization results are included in the repository. No LAMMPS
installation or HPC system is required.

### Setup

```bash
# From the repo root
pip install -e .
pip install cmcrameri          # optional but recommended
pip install tomli              # only needed for Python < 3.11
```

### Generate plots

**GA convergence comparison across materials:**

```bash
python scripts/plot_ga_comparison.py --boundary sigma5_310_STGB \
    --materials Ni Fe
# Optional: fix the inset y-axis upper bound
python scripts/plot_ga_comparison.py --boundary sigma5_310_STGB \
    --materials Ni Fe --inset-y-hi 1.25
```

**Per-material MC band plots**:

```bash
python scripts/plot_mc_comparison.py --material Fe --boundary sigma5_310_STGB
python scripts/plot_mc_comparison.py --material Ni --boundary sigma5_310_STGB
# For a different acceptance energy:
python scripts/plot_mc_comparison.py --material Fe --boundary sigma5_310_STGB \
    --e-accept 0.01
```

## Track 2 — Re-run the optimization

### Requirements

- LAMMPS compiled with the KOKKOS and MANYBODY packages. The results in
  the manuscript were produced with LAMMPS version `patch_30Mar2026`
  on INL's Teton cluster.
- A SLURM-based HPC cluster.
- The `slurm_utils.py` module (included in `scripts/`) provides job
  submission and monitoring utilities used by the run scripts.

> **Note:** The SLURM scripts generated by `run_ga.py` contain
> cluster-specific settings (partition names, `wckey`, module load
> commands, MPI launcher flags) that were used on Teton. You will need
> to adapt these for your own cluster before submitting jobs.
> `run_mc.py` invokes `srun` directly using the same settings read from
> `config/mc.toml`.

### Generate initial structures

```bash
python scripts/make_initial.py --material Fe --boundary sigma5_310_STGB
```

### Run GA optimization

Each run is written to `{material}/{boundary}/GA/{standard|high_energy}/run{N}/`.
The random seed is derived automatically as `seed + (N - 1)`, where `seed` is set
in `config/ga.toml`. To override any parameter on a per-run basis, place an
`overrides.toml` file in the run directory before launching — it is deep-merged
over `ga.toml` at startup.

```bash
# Standard initialization
python scripts/run_ga.py --material Fe --boundary sigma5_310_STGB \
    --run 1

# High-energy restart (requires a completed standard run)
python scripts/run_ga.py --material Fe --boundary sigma5_310_STGB \
    --run 1 --high-energy
```

### Run MC optimization

Each run is written to `{material}/{boundary}/MC/{standard|high_energy}/run{N}/`.
The seed is derived the same way as for GA (`seed + (N - 1)`, base in `config/mc.toml`).
An `overrides.toml` in the run directory is also supported.

```bash
python scripts/run_mc.py --material Fe --boundary sigma5_310_STGB \
    --run 1 --e-accept 0.1
```

### Extract GA data for plotting

After GA runs complete, extract the lightweight JSON summaries used by
the plotting scripts (this replaces the large `pickled_ga_object.pkl`
files which are not committed to the repository):

```bash
python scripts/extract_ga_data.py
```

## Boundaries

The following boundaries are included:

| Name | Σ | Misorientation axis | Angle | Type |
|------|---|---------------------|-------|------|
| `sigma3_coherent_twin` | 3 | [111] | 60° | Symmetric tilt |
| `sigma5_310_STGB` | 5 | [001] | 36.87° | Symmetric tilt |
| `sigma5_twist` | 5 | [001] | 36.87° | Twist |
| `sigma5_210_11_2bar_0_ATGB` | 5 | [001] | 36.87° | Asymmetric tilt |
| `sigma5_mixed` | 5 | [001] | 36.87° | Mixed |
| `sigma13_510_STGB` | 13 | [001] | 22.62° | Symmetric tilt |

## Literature reference values

Reference GBE values used for comparison are stored in
`config/literature_gbe.toml` and are sourced from:

- Fe (BCC): Ratanaphan *et al.*, Acta Materialia 88, 2015
- Ni (FCC): Olmsted *et al.*, Acta Materialia 57, 2009
